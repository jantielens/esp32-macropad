Binding Reference

Binding syntax

Use bindings in labels, colors, widget inputs, and wake expressions. Static text around tokens is preserved.

Overview Format Fallback MQTT Health Time Expr Pad Widgets

Common snippets

Start here for the patterns you use most often.

MQTT raw value 
[mqtt:sensors/temp]

Subscribe to a topic and display the raw payload.

Formatted MQTT JSON 
[mqtt:sensors/state;temperature;%.1fC]

Read a JSON field and format it inline.

Health metric 
[health:cpu]

Show a device metric like CPU, RSSI, or free heap.

Clock 
[time:%H:%M;Europe/Amsterdam]

Render local time with an Olson timezone.

Threshold color 
[expr:threshold([mqtt:sensor;temp],"#16a34a",25,"#f59e0b",35,"#dc2626")]

Map a number to a color without nested ternaries.

Pad reuse 
[pad:power;%.0f W]

Reuse a page-level binding across multiple buttons.

Multiple bindings 
[mqtt:solar;power;%.0f] / [mqtt:battery;soc;%.0f%%]

Combine several tokens in one label — static text between them is preserved.

Format strings

The optional format parameter is a printf-style specifier. Embed units directly inside it.

[scheme:…;%.2f unit]
%d
Integer — 42
%f
Float, full precision — 3.141593
%.1f
1 decimal place — 3.1
%.2f
2 decimal places — 3.14
%.0f
Rounded to integer — 3
%05d
Zero-padded to 5 chars — 00042
%s
String — pass-through
%%
Literal percent sign — %

Supported by: mqtt, health, expr, pad. Not used with time (uses strftime syntax instead) or in widget data-binding fields (those need a raw number, not formatted text).

Temperature with unit 
[mqtt:sensors/state;temperature;%.1f °C]

Unit embedded in the format — no suffix needed outside the token.

Integer with unit 
[mqtt:grid/state;power;%d W]

Drop decimals from whole-number readings.

Percentage sign 
[health:cpu;%.0f%%]

Use %% in the format string to output a literal %.

Two decimal precision 
[mqtt:energy;cost;%.2f €]

Use %.2f for currency or high-precision sensors.

Zero-padded counter 
[health:uptime;%05d s]

Fixed-width output so the label does not shift width.

Format after expression 
[expr:[mqtt:solar;power] / 1000;%.2f kW]

Format applies after the expression is evaluated.

Fallback values

Override the default --- placeholder when a binding has not resolved yet.

[scheme:params|fallback]
Numeric startup default 
[mqtt:solar/power;watts|0]

Use 0 while MQTT is still connecting.

Color-safe fallback 
[mqtt:light/color;hex|#f8fafc]

Use a valid color value instead of --- for color-capable fields.

MQTT bindings

Subscribe to a topic, optionally extract a JSON path, and optionally format the result.

[mqtt:topic;path;format]
topic
MQTT topic to subscribe to. Required.
path
JSON field to extract. Use . or leave it empty for the raw payload.
format
Printf-style output like %d, %.1f, or %s. Omit for raw text.
Raw payload 
[mqtt:sensors/temp]

Use when the topic already contains the exact value you want.

JSON field with formatting 
[mqtt:sensors/state;temperature;%.1fC]

Extract temperature from JSON and format as a single decimal.

Text around a binding 
Power: [mqtt:grid;power;%d] W

Static prefix and suffix are preserved.

Health bindings

Read device telemetry without MQTT.

[health:key;format]

cpu
CPU usage — 0–100 (integer)
uptime
Seconds since boot (integer)

Free heap bytes
heap_min
Lowest free heap since boot
heap_largest
Largest free heap block
heap_internal
Free internal (non-PSRAM) heap
psram_free
Free PSRAM bytes
psram_min
Lowest free PSRAM since boot
psram_largest
Largest free PSRAM block

WiFi

wifi_connected
ON / OFF
wifi_ssid
Connected network name (or ?)
rssi
Signal strength in dBm (integer)
ip
Device IP address
hostname
mDNS hostname

BLE (ESP32-P4 only)

ble_status
disabled, ready, pairing, connected, or error

CPU with suffix 
CPU: [health:cpu]%

Keep the unit outside the token if you do not need printf formatting.

Formatted heap 
[health:heap_free;%d B]

Format a raw numeric metric with a unit in one step.

WiFi status label 
[health:wifi_ssid] ([health:rssi] dBm)

Show connected network name and signal strength.

WiFi status color 
[expr:[health:wifi_connected]=="ON"?"#16a34a":"#dc2626"]

Use as a label or icon color — green when connected, red when not.

BLE status 
[health:ble_status]

Compact user-facing BLE state.

BLE status color 
[expr:[health:ble_status]=="connected"?"#16a34a":[health:ble_status]=="pairing"?"#f59e0b":[health:ble_status]=="ready"?"#3b82f6":[health:ble_status]=="error"?"#dc2626":"#64748b"]

Map BLE status to a color: green connected, amber pairing, blue ready, red error, gray disabled.

Time bindings

Render time with standard strftime tokens plus sub-second helpers.

[time:format;timezone]

Supported extras: %ms, %cs, %ds, and standalone %ums for uptime milliseconds. Use Olson names like Europe/Amsterdam or a POSIX timezone string.

Local time 
[time:%H:%M;Europe/Amsterdam]

Show local wall clock time with automatic DST handling.

Sub-second clock 
[time:%H:%M:%S.%ms]

Useful for debug labels or fast expressions.

Uptime in milliseconds 
[time:%ums]

Works even before NTP has synced.

Expression bindings

Combine MQTT, health, time, and pad bindings with arithmetic, comparisons, ternaries, and threshold().

[expr:expression;format]

Inner bindings resolve first. Use this for derived values, label text, visibility logic, and dynamic colors.

Unit conversion 
[expr:[mqtt:sensors/temp;value] * 9/5 + 32;%.1f]

Convert Celsius to Fahrenheit before formatting.

Conditional text 
[expr:[health:cpu] > 80 ? "High" : "Normal"]

Return strings directly from a condition.

Threshold color map 
[expr:threshold([mqtt:sensor;temp],"#16a34a",25,"#f59e0b",35,"#dc2626")]

Better than nested ternaries for 3+ color zones.

Fast blinking alert 
[expr:[mqtt:energy/power] > 2.5 ? ([time:%ms] % 500 < 250 ? "#dc2626" : "#f8fafc") : "#0f172a"]

Flash a color only while a live threshold is exceeded.

Pad bindings Pads only

Reuse a page-level binding by name instead of repeating the full source expression.

[pad:name;format]
Named binding 
[pad:power]

Use the raw value from the page-level bindings list.

Formatted pad binding 
[pad:power;%.0f W]

Apply a local format without redefining the source binding.

Pad binding inside expr 
[expr:[pad:power] > 3000 ? "High" : "Low"]

Good for status text or state logic based on reused inputs.

Widget numeric bindings

Data-binding fields for widgets expect a number after resolution.

Do not include a format parameter in widget data bindings. Formatting turns the value into display text and breaks numeric parsing.

MQTT JSON number 
[mqtt:home/energy/solar;power]

Feed a bar chart, gauge, or sparkline directly from JSON.

Health metric 
[health:cpu]

Useful for local device monitoring widgets.

Derived numeric value 
[expr:[mqtt:solar;w] - [mqtt:grid;w]]

Expressions are valid as long as they still resolve to a number.
